Tâche 3 : Cas d'utilisation

Cas d'utilisation : S’inscrire comme acheteur/ revendeur
But : avoir un compte pour acheter ou vendre sur la plateforme Unishop
Préconditions : avoir accès à la plateforme Unishop.
Acteurs : L’utilisateur est l’acteur principal
Scénario principal :
1. L’utilisateur ouvre la plateforme Unishop
2. L’utilisateur sélectionne l’option créer un compte
3. L’utilisateur sélectionne l’option créer un compte acheteur
4. L’utilisateur entre dans les cases indiquées son nom, prénom, pseudo, adresse courriel et téléphone
5. L’utilisateur entre son adresse d’expedition
6. Le client clique sur créer un profil
7. Le système valide son profil

Scénarios alternatifs :
3.a.1 L’utilisateur sélectionne l’option créer un compte revendeur
3.a.2 L’utilisateur entre dans les cases indiquées son nom, adresse, adresse courriel et téléphone
3.a.3 le scenario reprends à l’étape 6

7.a.1 Si une des information entrée n’est pas valide le système ne valide pas le profil et envoie un message d’erreur à l’utilisateur
7.a.2 Le scenario reprends à l’étape 4

Postconditions : Le profil et les informations qui lui sont liées sont enregistrées dans le système.

Cas d'utilisation : Passer commande
But : acheter des items sur la plateforme Unishop
Préconditions : avoir accès à la plateforme Unishop et avoir un compte.
Acteurs : L’utilisateur est l’acteur principal, le revendeur est acteur secondaire
Scénario principal :
1. L’utilisateur ouvre la plateforme Unishop
2. L’utilisateur se connecte
3. L’utilisateur ouvre son panier
4. L’utilisateur selectionne l'option commander
5. L’utilisateur selectionne les informations déjà sauvegardées sur son profil pour son adresse et son numéro de téléphone
6. L'utilisateur entre ses informations de payement et de livraison.
7. Le client confirme la commande
8. Le système valide la commande
9. Le système crée un identifiant unique pour la commande qu'il communique à l'utilisateur.
10. Le système mets à jour la quantité dans l'inventaire du revendeur.
11. Le systeme crée une page de suivi de la commande

Scénarios alternatifs :
4.a.1 Si le panier est vide le système envoie un message d'erreur au client

5.a.1 L’utilisateur choisit de rentrer de nouvelles informations
5.a.2 L'utilisateur entre son adresse de livraison
5.a.3 Le scénario reprends à l'étape 4

8.a.1 Si les informations rentrées sont fausses le système envoie un message d'erreur à l'utilisateur
8.a.2 Le scénario reprends à l'étape 5

Postconditions : La commande et les informations qui lui sont liées sont enregistrées dans le système.

Cas d'utilisation : Rechercher un produit / revendeur
But : trouver des items ou un revendeur spécifique sur la plateforme Unishop
Préconditions : avoir accès à la plateforme Unishop.
Acteurs : L’utilisateur est l’acteur principal
Scénario principal :
1. L’utilisateur ouvre la plateforme Unishop
2. L'utlisateur va dans la barre de recherche
3. L'utilisateur entre les mot-clés ou un pseudo de revendeur souhaités dans la barre
4. L'utilisateur appuie sur la touche entrer ou sur l'icone pour lancer la recherche
5. Le site affiche les résultats correspondant à la recherce de l'utilisateur
Scénarios alternatifs :
5.a.1 Si le site ne trouve aucun arcticle ou revendeur correspondant à la recherche de l'utilisateur, il renvoie une erreur
5.b.1 Si l'utlisateur n'a rien saisie dans la barre de recherche le site ne retourne rien
Postconditions :

Cas d'utilisation : Consulter / modifier état de commande
But : consulter ou modifier l'état d'une commande sur la plateforme Unishop
Préconditions : avoir accès à la plateforme Unishop, avoir un compte et avoir passé un commande
Acteurs : L’utilisateur et le revendeur sont les acteurs principaux. Le revendeur, le système du site et la compagnie de livraison sont les acteurs secondaires.
Scénario principal :
1. L’utilisateur ouvre la plateforme Unishop
2. L'utlisateur va dans l'onglet "mes commandes"
3. L'utilisateur clique sur la commande qu'il souhaite consulter
4. L'utilisateur clique sur "consulter l'état"
5. Le site affiche l'état de sa commande
6. L'utilisateur clique sur "modifier l'état de la commande" 7. L'utilisateur sélectionne le statut actuel de la commande Scénarios alternatifs :
6.a.1 Si l'acheteur a confirmer la reception de la commande, la modification n'est plus possible
7.a.1 L'utlisateur ne peut pas mettre la commande dans un état qui a été validé précédemment
Postconditions :Le système enregistre l'état de la commande une fois qu'il a été actualisé

Cas d'utilisation : Modifier profil
But : modifier les informations que la plateforme a à propos de l'utilisateur
Préconditions : avoir accès à la plateforme Unishop et avoir un compte.
Acteurs : L’utilisateur est l’acteur principal
Scénario principal :
1. L’utilisateur ouvre la plateforme Unishop
2. L’utilisateur se connecte
3. L’utilisateur selectionne son profil
4. L’utilisateur selectionne l'option modifier le profil
5. L’utilisateur est un acheteur il a donc le choix de modifier dans les cases indiquées son nom, prénom, pseudo, adresse courriel, son téléphone ou son adresse d'expédition
6. Le client clique sur modifier le profil
7. Le système valide son profil

Scénarios alternatifs :
5.a.1 L’utilisateur est un revendeur il peut donc modifier dans les cases indiquées son nom, adresse, adresse courriel et téléphone
7.a.1 Si une des information entrée n’est pas valide le système ne valide pas le profil et envoie un message d’erreur à l’utilisateur
7.a.2 Le scenario reprends à l’étape 5

Postconditions : Les nouvelles informations du profil remplacent les précédentes dans le système.

Cas d'utilisation : Voir métriques
But : Pouvoir voir les chiffres de son activité ou de produits sur le site
Préconditions : avoir accès à la plateforme Unishop et avoir un compte.
Acteurs : L’utilisateur est l’acteur principal
Scénario principal :
1. L’utilisateur ouvre la plateforme Unishop
2. L’utilisateur se connecte
3. L’utilisateur sélectionne son profil
4. L’utilisateur sélectionne l'option voir métriques
5. L’utilisateur voit les données liées à ses activités

Scénarios alternatifs :
3.a.1 L’utilisateur est un acheteur et veut voir les données d'un produit, il effectue donc une recherche pour ce produit
3.a.2 L'utilisateur clique sur le produit dont il veut voir les metriques.
3.a.3 Les métriques du produit sont affichées et facilement visible.

Postconditions : Le système est dans le même état qu'avant d'afficher les métriques.
